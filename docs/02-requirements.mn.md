## 1. Танилцуулга

### 1.1 Зорилго

Энэхүү баримт бичиг нь Ногоолин платформын функциональ болон функциональ бус шаардлагуудыг тодорхойлно. Энэ нь систем юу хийх ёстойг дизайн, мэдээллийн сангийн загварчлал, API тодорхойлолт, хэрэгжүүлэлт эхлэхээс өмнө тогтоосон эрх мэдэлтэй лавлагаа болно.

### 1.2 Хамрах Хүрээ

Энэхүү баримт бичиг дараах зүйлсийг хамарна:
- Нийтийн вэб каталог (Next.js)
- Администраторын самбар (Next.js `/admin`)
- Flutter мобайл аппликейшн
- Express.js REST API backend
- Supabase PostgreSQL өгөгдлийн давхарга

### 1.3 Шаардлагын Нэрлэх Дүрэм

Шаардлага бүр өвөрмөц кодоор тодорхойлогдоно:

```
FR-[МОДУЛЬ]-[ДУГААР]   → Функциональ Шаардлага
NFR-[МОДУЛЬ]-[ДУГААР]  → Функциональ Бус Шаардлага
```

**Чухалчлалын түвшин (MoSCoW):**

| Чухалчлал | Шошго | Утга |
|---|---|---|
| **M** | Заавал байх ёстой | MVP-д заавал шаардагдана. Үүнгүйгээр нэвтрүүлэх боломжгүй. |
| **S** | Байвал сайн | Чухал боловч нэвтрүүлэлтийг хааж чадахгүй. |
| **C** | Байж болох | Байвал сайн. Хугацаа дутвал хойшлуулна. |
| **W** | Хийхгүй | Одоогийн хамрах хүрээнд огт орохгүй. |

## 2. Оролцогч Талууд

| Оролцогч тал | Үүрэг | Гол ашиг сонирхол |
|---|---|---|
| Зочин / Хэрэглэгч | Нийтийн вэб эсвэл апп хэрэглэгч (нэвтрээгүй) | Бүртгэлгүйгээр бүтээгдэхүүнийг чөлөөтэй үзэх |
| Хэрэглэгч | Нэвтэрсэн хэрэглэгч | Хүсэлт гаргах, түүхийг харах (ирээдүй: захиалга хийх) |
| Администратор | Бизнесийн эзэн / оператор | Каталог удирдах, хүсэлт харах, хүргэлт асаах/унтраах |
| Хүргэгч | Ирээдүйн үүрэг | Хуваарилагдсан захиалгыг харах, статус шинэчлэх |
| Хөгжүүлэгч | Нэг хөгжүүлэгч (Тэнгис) | Засвар үйлчилгээ хийж болох, баримт бичигтэй, тестлэж болох кодын бааз |
