### UC-G-001 — 3D Нээлтийн Анимацийг Үзэх эсвэл Алгасах

| Талбар | Утга |
|---|---|
| **Оролцогч** | Зочин |
| **Өдөөгч** | Зочин анх удаа үндсэн URL `/`-д хандана |
| **Урьдчилсан нөхцөл** | Зочин сайтад өмнө нь зочлоогүй (эсвэл сессийг цэвэрлэсэн) |
| **Дараагийн нөхцөл** | Зочин нүүр хуудсанд очно |

**Үндсэн Урсгал:**
1. Зочин `/`-д хандана
2. Систем 3D GLB загварыг дэвсгэрт ачааллаж эхэлнэ
3. Систем загвар ачаалагдаж байх үед ачаалалтын индикатор харуулна
4. Систем 3D нээлтийн анимаци тоглуулна (Ногоон Дарь Эхийн загвар, дугуй камерын хөдөлгөөн)
5. Анимаци дуусна; Систем "Эхлэх" товч харуулна
6. Зочин "Эхлэх" товч дарна
7. Систем зочиныг нүүр хуудас руу чиглүүлнэ

**Альтернатив Урсгал А — Алгасах:**
- 3 эсвэл 4-р алхамд зочин "Алгасах" товч дарна
- Систем зочиныг шууд нүүр хуудас руу чиглүүлнэ

**Альтернатив Урсгал Б — Буцаж Ирсэн Зочин:**
- Зочин өмнө нь зочилсон (сессийн тугаар тэмдэглэгдсэн)
- Систем анимацийг алгасаж шууд нүүр хуудас руу чиглүүлнэ

**Альтернатив Урсгал В — Хөдөлгөөн Багасгасан Тохиргоо:**
- Зочны үйлдлийн систем `prefers-reduced-motion` идэвхжүүлсэн
- Систем анимацийг алгасаж статик hero зургийн дээр "Эхлэх" товчийг шууд харуулна

**Онцгой Урсгал — WebGL Боломжгүй:**
- Систем WebGL дэмжигдэхгүй эсвэл төхөөрөмжийн гүйцэтгэл хангалтгүй гэдгийг илрүүлнэ
- Систем 3D загварын оронд статик fallback зураг харуулна
- Систем "Эхлэх" товчийг шууд харуулна

**Альтернатив Урсгал — Дууссан Нөөц:**
- Бүтээгдэхүүний `stock_status` нь `out_of_stock`
- Систем нөөцийн статусын badge харуулна; хүсэлтийн маягт ашиглах боломжтой хэвээр байна

### UC-G-005 — 360° 3D Бүтээгдэхүүний Загварыг Харах

| Талбар | Утга |
|---|---|
| **Оролцогч** | Зочин |
| **Өдөөгч** | Бүтээгдэхүүний дэлгэрэнгүй хуудас ачаалагдаж, бүтээгдэхүүнд `model_3d_url` байна |
| **Урьдчилсан нөхцөл** | Зочин бүтээгдэхүүний дэлгэрэнгүй хуудсанд байна; `model_3d_url` байна; WebGL боломжтой |
| **Дараагийн нөхцөл** | Зочин 3D загвартай харилцсан байна |

**Үндсэн Урсгал:**
1. Систем `model_3d_url`-аас (Supabase Storage CDN) GLB файлыг ачааллана
2. Систем загвар татагдаж байх үед ачаалалтын индикатор харуулна
3. Систем Three.js OrbitControls-тай харагчид (вэб) эсвэл `model_viewer_plus`-т (мобайл) загварыг зурна
4. Зочин чирэх замаар (хулгана / хүрэлт) загварыг эргүүлнэ
5. Зочин scroll хүрд эсвэл pinch дохиогоор томруулна
6. Зочин баруун товч чирэх (вэб) эсвэл хоёр хуруу чирэх (мобайл) замаар шилжүүлнэ

**Альтернатив Урсгал — 3D Загвар Байхгүй:**
- `model_3d_url` null байна
- Систем 3D харагчийг зурахгүй
- Систем зургийн gallery-г үндсэн visual болгон харуулна

**Онцгой Урсгал — GLB Ачаалах Амжилтгүй:**
- GLB файл ачаалах амжилтгүй болно (сүлжээний алдаа, гэмтсэн файл)
- Систем зургийн gallery руу чимээгүй буцна
- Систем алдааг бүртгэнэ

### UC-G-007 — Бүтээгдэхүүний Хүсэлт Гаргах

| Талбар | Утга |
|---|---|
| **Оролцогч** | Зочин |
| **Өдөөгч** | Зочин бүтээгдэхүүний дэлгэрэнгүй хуудасны "Хүсэлт гаргах" товч дарна |
| **Урьдчилсан нөхцөл** | Зочин бүтээгдэхүүний дэлгэрэнгүй буюу холбоо барих хуудсанд байна |
| **Дараагийн нөхцөл** | Хүсэлт `new` статустайгаар хадгалагдана; зочин баталгаажуулалт хүлээн авна |

**Үндсэн Урсгал:**
1. Зочин хүсэлт гаргах товч дарна
2. Систем хүсэлтийн маягт харуулна: нэр, утасны дугаар, мессеж, урьдчилан бөглөгдсөн бүтээгдэхүүний лавлагаа
3. Зочин маягтын талбаруудыг бөглөнө
4. Зочин маягтыг илгээнэ
5. Систем бүх шаардлагатай талбарыг client талаас шалгана
6. Систем `POST /api/v1/inquiries`-г маягтын өгөгдөлтэйгээр илгээнэ
7. Систем талбаруудыг сервер талаас шалгаж rate limit-ийг шалгана (IP тус бүр цагт 3)
8. Систем хүсэлтийг `new` статустайгаар мэдээллийн санд хадгална
9. Систем зочинд амжилтын баталгаажуулалтын мессеж харуулна

**Альтернатив Урсгал — Шалгалтын Алдаа:**
- Шаардлагатай талбарууд хоосон эсвэл утасны дугаарын формат буруу байна
- Систем алдаатай талбаруудыг алдааны мессежтэйгээр тодруулна
- Зочин талбаруудыг засаж дахин илгээнэ

**Онцгой Урсгал — Rate Limit Хэтэрсэн:**
- Зочин ижил IP-аас сүүлийн нэг цагт 3 хүсэлт илгээсэн байна
- Систем HTTP 429 буцаана
- Систем мессеж харуулна: "Хэт олон хүсэлт. Дараа дахин оролдоно уу."

|---|---|

6. Систем сагсны зүйл бүрд `order_items` бүртгэл үүсгэнэ
7. Систем захиалгын ID-тай захиалгын баталгаажуулалтыг буцаана
8. Систем Администраторт шинэ захиалгыг мэдэгдэнэ (ирээдүй: push мэдэгдэл эсвэл email)
